## 📱 项目概述

### 项目定位
**EchoSage** 是一款基于 **uniapp + Vue3** 开发的智能英语语音对话学习应用，通过模拟真实对话场景，帮助用户提升英语口语能力。

### 核心特色
1. **双模式交互**
   - **普通对话模式**: 传统聊天界面，支持文本输入和语音输入，用户可手动点击播放AI语音回复
   - **电话练习模式**: 模拟真实电话通话，全语音交互，AI回复自动播放，更接近真实对话场景

2. **AI驱动对话**
   - 集成后端AI服务（Flask）
   - 支持语音识别（Speech-to-Text）
   - 支持语音合成（Text-to-Speech）
   - 智能对话生成，根据上下文生成回复

3. **完整的对话管理**
   - 对话历史记录
   - 模式切换历史继承
   - 历史对话恢复
   - 开启新对话功能

### 2. 对话模块 (pages/chat/chat.vue)

**核心页面**: 应用的心脏，承载所有对话交互

#### 2.1 消息管理
```javascript
// 消息数据结构
{
id: string,              // 唯一标识
content: string,         // 消息内容
isUser: boolean,         // 用户/AI标识
timestamp: number,       // 时间戳
status: string,          // 'sending' | 'sent' | 'failed'
type: string,            // 'text' | 'voice'
duration?: number,       // 语音时长
audioUrl?: string,       // 音频URL
filePath?: string        // 本地文件路径
}
```

#### 2.2 录音管理
- **普通模式**: 录音由 `NormalInput` 组件管理
- **电话模式**: 录音由页面直接管理（uni.getRecorderManager）
- 录音配置:
- 格式: WAV
- 采样率: 16000Hz
- 单声道
- 最长60秒

#### 2.3 历史记录管理
**存储策略**:
```javascript
// 存储键（按模式分开）
'echosage_history_normal'  // 普通模式历史
'echosage_history_phone'   // 电话模式历史

// 历史记录结构
{
id: string,
mode: 'normal' | 'phone',
timestamp: number,
messages: Message[]
}
```

**核心逻辑**:
- ✅ 自动保存：页面卸载时保存
- ✅ 去重检测：避免重复保存相同对话
- ✅ 更新机制：恢复的对话更新时不创建新记录
- ✅ 限制数量：每个模式最多保留50条历史
- ✅ 模式切换：切换时删除旧模式中的当前对话ID记录

#### 2.4 模式切换
**快速切换流程**:
```
1. 停止录音（如有）
2. 保存当前对话到旧模式
3. 删除旧模式中当前对话ID记录
4. 切换到新模式
5. 重置对话状态（currentHistoryId = null）
6. 添加切换提示消息
7. 停止音频播放
8. 显示切换提示
```

## 📊 数据流设计

### 1. 文本对话流程

```
用户输入文本
↓
handleSendText(text)
↓
添加用户消息到 messages
↓
调用 sendChatMessage(text, history)
↓
后端 API: POST /api/chat
↓
返回 { text, audio_url, metadata }
↓
添加AI消息到 messages（包含 audioUrl）
↓
滚动到底部
↓
标记 hasNewMessages = true
```

### 2. 语音对话流程

```
用户按录音按钮
↓
startVoiceRecording()
↓
uni.getRecorderManager().start()
↓
录音中... (计时器更新时长)
↓
用户松开/停止录音
↓
onStop → 获取 tempFilePath
↓
读取文件转 base64
↓
调用 speechToText(base64, 'wav')
↓
后端 API: POST /api/speech-to-text
↓
返回识别文本
↓
更新语音消息内容
↓
调用 sendChatMessage(recognizedText, history)
↓
后端 API: POST /api/chat
↓
返回 AI 回复（带 audio_url）
↓
添加AI消息
↓
普通模式: 用户点击播放
电话模式: 自动播放（未完全实现）
```

### 3. 历史记录保存流程

```
触发条件:
- 页面卸载 (onUnmounted)
- 模式切换 (switchMode)
↓
saveHistory()
↓
检查 hasNewMessages 标志
↓
[是] → 继续保存流程
[否] → 跳过保存
↓
检查 currentHistoryId
↓
[有ID] → 更新现有记录
[无ID] → 创建新记录
↓
去重检测（比较消息内容）
↓
保存到 localStorage
↓
限制最多50条记录
```

## 🌐 后端集成

### API 配置

**后端地址**: `http://192.168.56.1:8000`

**核心端点**:

| 端点 | 方法 | 功能 | 输入 | 输出 |
|------|------|------|------|------|
| `/api/chat` | POST | 文本对话 | message, history, mode | text, audio_url, metadata |
| `/api/speech-to-text` | POST | 语音识别 | audio (base64), format | text, confidence, language |
| `/api/text-to-speech` | POST | 语音合成 | text, language | audio_url, duration, size |
| `/api/audio/<file_id>` | GET | 音频文件 | file_id | 音频文件流 |

### API 客户端特性

```javascript
// utils/api.js 提供的功能

✅ 统一请求封装 (request方法)
✅ 自动重试机制 (最多3次，延迟1秒)
✅ 超时控制 (50秒)
✅ 错误处理与转换
✅ 请求日志输出
✅ 便捷方法导出:
- speechToText(audioBase64, format)
- sendChatMessage(message, history, options)
- textToSpeech(text, options)
- uploadFile(filePath, options)
```

### 数据格式

**请求格式示例**:
```json
// POST /api/chat
{
"message": "Hello, how are you?",
"history": [
{ "role": "user", "content": "Hi" },
{ "role": "assistant", "content": "Hello!" }
],
"mode": "normal"
}
```

**响应格式示例**:
```json
{
"success": true,
"data": {
"text": "I'm doing great, thank you!",
"audio_url": "/api/audio/tmp_abc123.mp3",
"metadata": {
"tokens_used": 25,
"processing_time": 1234.56,
"language_detected": "en"
}
},
"message": "处理成功"
}
```

## 💾 本地存储设计

### 存储键清单

| 键名 | 用途 | 数据结构 | 保存时机 |
|------|------|---------|---------|
| `echosage_history_normal` | 普通模式历史 | HistoryItem[] | 页面卸载/模式切换 |
| `echosage_history_phone` | 电话模式历史 | HistoryItem[] | 页面卸载/模式切换 |
| `echosage_selected_history` | 临时选中历史 | { mode, history } | 点击历史记录时 |
| `echosage_current_mode` | 当前模式 | 'normal' \| 'phone' | 模式切换时 |
| `echosage_audio_settings` | 音频设置 | { volume, speed, muted } | 设置更改时 |

```typescript
// 历史记录项
interface HistoryItem {
id: string                    // 时间戳字符串

}

// 消息对象
interface Message {
id: string                    // 消息ID
content: string               // 内容
isUser: boolean               // 是否用户
timestamp: number             // 时间戳
status: 'sending' | 'sent' | 'failed'
type: 'text' | 'voice'
duration?: number             // 语音时长
audioUrl?: string             // 音频URL
filePath?: string             // 本地路径
}
```
